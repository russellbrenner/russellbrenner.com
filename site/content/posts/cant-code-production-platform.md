I've never held a software engineering title. Not once in 25 years.

My career is networking, pre-sales, solutions architecture. I started at an ISP in Melbourne in 2000. Worked my way through carrier ethernet, professional services, enterprise networking. Sold software-defined networking during the NFV wave. Managed security architecture for cloud providers. None of those jobs required me to write production code.

Today I'm operating:

- 114 Kubernetes deployments across 80 namespaces
- A multi-agent orchestration platform with 13 custom agent definitions
- 50+ MCP tools spanning secrets, source control, project management, and legal research
- Seven-layer monitoring with automated remediation
- A legal RAG API with domain-specific embeddings

All in production. All AI-assisted.

The code is written by Claude Code. I architect the workflow, review the output, debug failures, and make the call on what gets built next. The agent writes the TypeScript, the Python, the React, the Svelte. I verify it works, deploy it, and operate it when it breaks at 2 AM.

This isn't a weekend project. It's 1,306 infrastructure commits in 2026 alone. Every change is a commit. Every service is manifest-driven. Every failure is my pager.

What changed wasn't my coding ability. I still can't code. What changed was that AI removed the ceiling on what one infrastructure person can build.

The full story of how 25 years of networking transferred into building production AI systems is in [I'm Not a Software Engineer](/posts/not-a-software-engineer). This post is the shorter version: I'm not a software engineer. I'm a future lawyer with 25 years of infrastructure judgment. The code is AI-assisted. The architecture, operations, and judgment are mine.

Both contributions are real.
